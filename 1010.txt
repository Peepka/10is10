Request Headers:

1. Accept-Language - Список поддерживаемых естественных языков.                                         (Accept-Language: ru)
2. Accept - Список допустимых форматов ресурса.                                                         (Accept: text/plain)
3. Authorization - 	Данные для авторизации.                                                             (Authorization: Basic QWxhZGRpbjpvcGVuIHNlc2FtZQ==)
4. Content-Disposition - Способ распределения сущностей в сообщении при передаче нескольких фрагментов. (Content-Disposition: form-data; name="MessageTitle")
5. From - Адрес электронной почты ответственного лица со стороны клиента.                               (From: user@example.com)
6. Host - Доменное имя и порт хоста запрашиваемого ресурса. Необходимо для поддержки виртуального хостинга на серверах. (Host: ru.wikipedia.org)
7. Expect - 	Указывает серверу что клиент ожидает от него дополнительного действия.                    (Expect: 100-continue)
8. Max-Forwards - Максимально допустимое количество переходов через прокси.                             (Max-Forwards: 10)
9. Proxy-Authorization - Информация для авторизации на прокси-сервере.                                  (Proxy-Authorization: Basic QWxhZGRpbjpvcGVuIHNlc2FtZQ==)
10. Range - Байтовые диапазоны для запроса фрагментов ресурса. Подробности: Частичные GET.              (Range: bytes=50000-99999,250000-399999,500000-)

Response Headers: 

1. Accept-Ranges - Перечень единиц измерения диапазонов.                                                (Accept-Ranges: bytes)
2. ETag - Тег (уникальный идентификатор) версии сущности, используемый при кэшировании.                 (ETag: "56d-9989200-1132c580")
3. Location - URI по которому клиенту следует перейти или URI созданного ресурса.                       (Location: http://example.com/about.html#contacts)
4. Server - 	Список названий и версий веб-сервера и его компонентов с комментариями. Для прокси-серверов поле Via. (Server: Apache/2.2.17 (Win32) PHP/5.3.5)
5. Vary - Список описывающих ресурс полей из запроса, которые были приняты во внимание.	                (Vary: Accept-Encoding)
6. Age - Количество секунд с момента модификации ресурса.
7. Alternates - Указание на альтернативные способы представления ресурса.
8. Proxy-Authenticate - Параметры аутентификации на прокси-сервере.
9. Public - Список доступных методов аналогично Allow, но для всего сервера.
10. Retry-After - Дата или время в секундах после которого можно повторить запрос.
